§ 0 · rappel

D'abord : c'est quoi un MCP server ?

MCP (Model Context Protocol) — un standard d'Anthropic adopté par Claude Code, Cursor, Manus, ChatGPT desktop. Au lieu de copier-coller des données dans ton agent, tu lui donnes une URL → l'agent appelle directement le serveur quand il a besoin d'une info.

Claude Code agent
→
openswissdata.com MCP server
→
réponse JSON en 200 ms

Exemple concret : tu tapes "trouve-moi le code TARES pour bijoux en or 18 carats" dans Claude Code → l'agent appelle openswissdata MCP → récupère le code 7113.19 avec disclaimer non-officiel et description trilingue → tu n'as rien à faire.

§ 6 · décisions critiques

5 questions pour valider V2 — illustrées

Chaque question a une recommandation (mon vote, en vert). Tu peux juste valider tout d'un coup avec la phrase recommandée tout en bas, ou ajuster question par question.

1

Quelle URL pour le serveur MCP ?

Question simple : où les agents (Claude Code, Cursor) iront chercher openswissdata. Aujourd'hui c'est sur le path /mcp du domaine principal. On peut le déplacer sur un sous-domaine dédié, plus pro pour la promo.

Option A · garder comme aujourd'hui
openswissdata.com/mcp
  • ✓ déjà en LIVE
  • ✓ rien à reconfigurer
  • — moins propre dans les docs
recommandé
Option B · sous-domaine dédié
mcp.openswissdata.com
  • ✓ plus court, plus pro
  • ✓ "Premier sous-domaine MCP de Suisse"
  • — ~10 min de DNS Infomaniak + Railway
2

Comment gérer les comptes / tokens ?

Pour facturer / limiter par utilisateur, il faut un système de "compte + token". C'est ce qu'on appelle l'OAuth 2.1. Deux choix : on l'écrit nous-même, ou on l'achète.

recommandé
Option A · in-house
🏠
On code nous-même avec lib oslo
  • Dev : 1.5 jour
  • Coût/mois : 0 CHF
  • Contrôle : total
  • Risque : faible (lib battle-tested)
Option B · outsource
☁️
Auth0 / Clerk / WorkOS
  • Dev : 0.5 jour
  • Coût/mois : 25-50 USD
  • Contrôle : dépend du provider
  • Risque : dépendance externe
3

STATENT (statistique entreprises) — on l'expose ou pas ?

Rappel : on a ingéré 978 614 lignes STATENT de l'OFS (combien d'établissements par commune × NOGA). Mais l'OFS demande une autorisation écrite avant vente commerciale (waiver), qu'on n'a pas encore.

recommandé
Option A · inclure GATED Pro
🔐
Disponible UNIQUEMENT pour acheteurs Pro

Comme c'est gated par OAuth scope, seuls les payeurs Pro y accèdent. Pas de redistribution publique → risque OFS minimal. On envoie le waiver en parallèle. Quand l'OFS répond, on le déclare officiel.

Option B · retirer du V2
⏳
On attend le waiver OFS écrit

Plus safe juridiquement (zéro risque). Mais l'OFS peut prendre des semaines à répondre, et on perd un argument fort pour la promo. On garde STATENT pour V3.

4

Combien de requêtes MCP par tier de prix ?

Combien d'appels MCP chaque utilisateur peut faire selon ce qu'il a payé. Plus on est généreux au tier Free, plus on attire de devs (= bouche à oreille). Plus le Pro est juteux, plus on capture de valeur.

Tier Quota Volume visuel Qui ?
Free 100 / jour
Devs qui testent / découvrent
Standard (inclus avec ZIP 299-399 CHF) 1 000 / mois
Acheteur du ZIP qui veut aussi une API

Pro user, accès STATENT, semantic search
Standalone (MCP seul, sans ZIP) 5 000 / mois — 49 CHF/mois

Mon vote : Validate ces chiffres tels quels — généreux Free pour évangéliser MCP, Pro juteux, Standalone à 49 CHF/mois pour capturer ceux qui veulent juste l'API.

5

Mode "reverse-call" — pour V2 ou plus tard ?

Reverse-call = un service externe (par ex. Strale qui vend sur x402) appelle openswissdata avant de servir un client, pour vérifier qui c'est. C'est le miroir de ce que fait Strale côté crypto.
