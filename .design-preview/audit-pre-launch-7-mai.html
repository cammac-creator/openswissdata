Audit 01 / Sécurité / Stack durcie, secrets propres, signing OK
🔒

Sécurité production

READY
  • ✅HTTPS + headers durcis : HSTS 180j + X-Frame DENY + COOP same-origin + Permissions-Policy (caméra/micro/géoloc OFF) — au-dessus de la moyenne SaaS
  • ✅Rate limiting actif : 1 succès puis 9× 429 sur /api/checkout/start et /mcp/oauth/register. Anti-spam OK.
  • ✅Endpoints admin protégés : 401 avec timing-safe compare contre ADMIN_SECRET
  • ✅Tier escalation impossible : client demande "pro" → forcé en "free" côté serveur. Promotion uniquement via webhook Stripe payé.
  • ✅Stripe webhook signé + idempotent + transaction atomique (pas de double order possible)
  • ✅Manifest Ed25519 vérifié OK : verify-provenance.ts retourne signatureValid=true + RFC-3161 freetsa
  • ✅Aucun secret leak dans le repo public : .env gitignoré, scan git log clean (placeholders only)
  • ✅CORS strict : pas d'Allow-Origin: * (Hono par défaut = same-origin)
  • 🟡CSP avec 'unsafe-inline' sur scripts/styles (commentaire dette dans le code) — à durcir post-launch via nonces (~2h)
  • 🟡Cookie session magic-link : flags non vérifiés en l'absence de credentials — à confirmer Secure/HttpOnly/SameSite
Audit 02 / Légal & Compliance / nLPD + RGPD + permissions sources
⚖️

Légal & Compliance B2B suisse

CONDITIONNEL
  • 🔴Disclaimer italien manquant (TARES) : seul "Non-officiel · Nicht-offiziell · Non-official" — manque "Non ufficiale". La permission BAZG exige les 3 langues officielles suisses. Bloquant légal.
  • 🔴Sentry + UptimeRobot non listés dans Privacy : manquement art. 19 nLPD + art. 13 RGPD (information sur les destinataires). Logs d'erreurs = données personnelles potentielles.
  • 🔴Statut RC < 100k CHF / non-TVA : légal mais friction commerciale B2B (départements achats refusent souvent les fournisseurs sans n° RC/TVA). Ajouter mention art. 36 ORC pour transparence.
  • 🟡FAQ retourne 404 : impossible de vérifier la cohérence de la promesse "refund 14j sans condition" entre CGV et FAQ.
  • 🟡OPDS non mentionné dans Compliance : à clarifier ou retirer pour ne pas créer de confusion en audit externe.
  • 🟡Téléphone absent Impressum : art. 322 CO + LCD exigent moyen de contact rapide. Email seul fragile en litige B2B.
  • ✅Provenance crypto + SDR Policy + Footer + CGV core dépassent les exigences légales

API + MCP — point critique du launch

NO-GO
  • 🔴statent_lookup MANQUANT : annoncé dans la spec "8 tools" mais retourne "Unknown tool" en prod. À la place, il y a finma_search non documenté. La promesse "8 tools" devient mensongère.
  • 🔴tariff_lookup retourne "No row found" pour HS8 84820010 alors que ce code est dans samples/tares-sample.csv ligne 1 (Roulements à billes). Le tool flagship démo plante sur le code le plus visible.

  • 🟡Samples publics trop petits : TARES 11 lignes (10 data + header) au lieu des 50-200 attendus. Démo peu convaincante.
  • ✅Health checks OK : /api/health/deep retourne DB 2ms, R2 110ms, Stripe 284ms — tous ok=true
  • ✅Protocol version + scope enforcement corrects : tariff_changelog sans auth → -32001 "Insufficient scope (requires 'tariff:history')"
